Lists posts belonging to the publication. Maps to GET /publications/{publicationId}/posts. Invoke via `node <base_directory>/beehiiv.js posts [flags]`. Output is JSON to stdout. This is the primary "search" command — Beehiiv has no search endpoint, so use filters + scanning titles to find posts.

[!FLAGS] a) no flags — returns up to 10 most recent posts. b) `--status <draft|confirmed|archived|all>` — filter by status (default: confirmed). c) `--content-tags <comma-list>` — filter by content tags (OR logic). d) `--authors <comma-list>` — filter by author names (case-insensitive, OR logic). e) `--audience <free|premium|all>` — filter by audience. f) `--platform <web|email|both|all>` — filter by platform. g) `--order-by <created|publish_date|displayed_date>` — sort field. h) `--direction <asc|desc>` — sort direction. i) `--limit <1-100>` — results per page (default 10). j) `--page <n>` — page number. k) `--expand <comma-list>` — expand fields: stats, free_web_content, free_email_content, free_rss_content, premium_web_content, premium_email_content. l) `--slugs <comma-list>` — filter by exact slugs. m) `--hidden-from-feed <all|true|false>` — filter by feed visibility. n) `--raw` — output the full API response envelope.

[!SEARCH-STRATEGY] To find a post by topic: run `posts --limit 100 --direction desc` to get recent posts, scan titles and subtitles. If not found, paginate with `--page 2`, etc. Once found, use `get <postId>` to read the full content.

[!OUTPUT-SHAPE] Default produces an array of post objects with id, title, subtitle, authors, created, status, publish_date, slug, web_url, audience, platform, content_tags. With `--raw`, wraps into the API envelope with `data`, `page`, `limit`, `total_results`, `total_pages`.
